# queueManager Library Analysis

## Overview
The `queueManager` library provides a type-safe, callback-driven queue for buffering and processing packets or objects. It is designed for use in embedded systems (ESP32/ESP8266/Arduino) where efficient, modular, and reliable packet management is required. The library leverages a ring buffer for storage and supports custom processing, filtering, and overflow handling via user-defined callbacks.

## Key Features
- **Type Safety:** Template-based for any packet/object type.
- **Power-of-Two Buffer:** Compile-time enforcement for optimal ring buffer performance.
- **Callback Architecture:** Supports processor, filter, and overflow callbacks for flexible queue management.
- **Efficient Buffering:** Uses a ringBuffer for fast, memory-efficient storage and retrieval.
- **Zero-Overhead Setters:** Template setters for callbacks, inlined for performance.
- **Overflow Handling:** Drops oldest packet on overflow and invokes user callback.
- **Integration:** Designed to work with packetProcessor, messageHandler, and other modular libraries.

## Main API
- `enqueue(const packetType&)`: Adds a packet to the queue (with filter and overflow logic).
- `dequeue(packetType&)`: Removes and returns the oldest packet.
- `processAll()`: Processes all packets using the processor callback.
- `setProcessor`, `setFilter`, `setOverflowHandler`: Set user callbacks.
- `isEmpty`, `isFull`, `size`, `capacity`, `clear`: Queue status and management.

## Usage Example
```cpp
#include <queueManager.hpp>
#include <deviceDataPacket.hpp>

queueManager<deviceDataPacket, 8> rxQueue;

void processPacket(const deviceDataPacket& pkt) {
    Serial.printf("Received opcode: 0x%02X\n", pkt.opcode);
}

void setup() {
    rxQueue.setProcessor(processPacket);
}

void loop() {
    rxQueue.processAll();
}
```

## Integration
- Works seamlessly with ringBuffer, packetProcessor, and messageHandler.
- Suitable for automaton, wireless, and device packet flows.

## Extensibility
- Use custom packet types and buffer sizes.
- Add advanced filtering or overflow logic as needed.

## Limitations
- Buffer size must be a power of two.
- No built-in thread safety (single-threaded embedded use).
